Bacula x OpenSSL 1.1

Enquanto a solução definitiva não sai, veja como contornar o problema

Posted by Marcus "Molinux" Molinero on July 28, 2017

O Problema

Essa semana tive a surpresa de ver alguns dos meus servidores dando erro no bacula-fd. Simplesmente pararam de funcionar e não subiam mais.

Descobri então que o problema estava com a atualização do OpenSSL para a versão 1.1, que até o momento dessa postagem, ainda é incompatível com o Bacula, como pode ser visto aqui.

E que o torna também incompatível com o Debian 9, como pode ser visto aqui.

A Solução

É possível ter 2 OpenSSL coexistindo na mesma máquina e para isso, vou partir do princípio que você tenha o mesmo cenário que o meu. O OpenSSL-1.1 instalado via pacote da sua distribuição e o Bacula compilado ( e não instalado via pacote ).

Instalaremos então a versão 1.0 em um local alternativo.

Baixando e instalando o OpenSSL-1.0.2l

cd /usr/src/
wget -c https://www.openssl.org/source/openssl-1.0.2l.tar.gz
tar xzvf openssl-1.0.2l.tar.gz
cd openssl-1.0.2l/
./config --prefix=/usr/local --openssldir=/usr/local/openssl
make
make test #passo opcional
make install

Recompilando e instalando o Bacula-FD com o OpenSSL-1.0.2l

cd ../bacula-7.4.7/
./configure --enable-client-only --with-systemd --with-openssl=/usr/local/openssl
make
make install
systemctl restart bacula-fd.service
